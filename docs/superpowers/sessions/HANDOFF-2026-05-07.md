## The S1.4 finding (action required before 3A/3B)

`--optimize-kernels` crashes on both OdinHalo AND OdinFlat with Sprint 1
enabled. Root cause:

```
autokernel/_patterns.py:314
_FusedResidualRMSNormBlockReplacement.forward(self, x, freqs_cis)
```

Does not accept `depth_kvs` (MoDA), `doc_mask` (intra-doc), `v_prev`
(value residuals), or `head_gate_active` (head gating). The pattern
matcher wraps any block with `(attn, pre_norm, ffn, ffn_norm)` attrs
via `_BLOCK_ALIASES`.

**Fix options (take your pick on return):**

1. **Quickest**: Add `_skip_autokernel = True` attribute on MoDA and
   value-residual blocks. Line 101 of `_patterns.py` already honors it:
   `if getattr(module, "_skip_autokernel", False): return None`
2. **Proper**: Extend `_FusedResidualRMSNormBlockReplacement.forward` to
   accept `**kwargs` and forward them to `self.attention`. Then detect
   which block classes have extra kwargs and either forward or raise.
3. **Heavy**: Custom replacement class for each block type
   (NoPEMoDAGQABlock, NoPEGQABlock). Safest but most code.

I recommend #1 first (1-line change on each block class; 5-min fix) +
measure throughput lift vs current baseline. If the lift is real and
worth the tech debt, bump to #2 later.
