### 5.2 SPECTRA implementation

SPECTRA (arXiv:2603.14315) provides two orthogonal clipping mechanisms, sequenced in our rollout (post first, then pre per brainstorm Q5 Option B).

#### Post-clipping (Phase A/C — core SPECTRA)

After the optimizer produces a weight-update tensor `ΔW`, clip its spectral norm to `clip_norm`:

```python
# halo_training/spectra.py
def apply_post_clip(update, clip_norm=1.0, ns_iterations=5):
    """Bound spectral norm of update via Newton-Schulz.

    For each 2D parameter update ΔW:
      M_orth ← msgn(ΔW) via Newton-Schulf (ns_iterations steps)
      spectral_norm ≈ largest singular value of ΔW  (from NS iteration's scaling factor)
      if spectral_norm > clip_norm:
        ΔW ← ΔW * (clip_norm / spectral_norm)
    Composite Frank-Wolfe with spectral-norm constraints.
    """
```

Invoked as a post-`step()` hook in `NorMuon.step()`:

```python
@torch.no_grad()
def step(self, closure=None):
    # ... standard NorMuon update computation into param.data ...
    if self.spectra_post:
        for param in muon_params:
            update = param.data.clone() - old_param_value
            clipped = apply_post_clip(update, self.clip_norm, self.ns_iterations)
            param.data.copy_(old_param_value + clipped)
```

For the AdamW group (1D / embeddings), post-clipping is not applied — SPECTRA targets matrix updates.

#### Pre-clipping (Phase D — secondary)

Before the optimizer step, clip the spectral norm of each parameter's gradient:

```python
# halo_training/spectra.py
def apply_pre_clip(model, clip_norm=1.0, ns_iterations=5):
    """Bound spectral norm of each 2D parameter's gradient after backward().

    Invoked between loss.backward() and optimizer.step():
      for each trainable 2D parameter p with p.grad:
        grad_spectral_norm = ... (via NS)
        if > clip_norm:
          p.grad.mul_(clip_norm / grad_spectral_norm)
    Suppresses sparse spectral spikes in gradient noise.
    """
```

Called from trainer explicitly between backward and step (same pattern as `torch.nn.utils.clip_grad_norm_` but spectral-aware).

#### CLI and toggling

Both modes independently toggleable. Default OFF until Phase C/D validate their contributions.

```
--spectra-post                     # post-clipping enabled
--no-spectra-post                  # post-clipping disabled
--spectra-pre                      # pre-clipping enabled
--no-spectra-pre
--spectra-clip-norm <float>        # spectral norm bound (default 1.0 until Phase B1 sweeps)
--spectra-ns-iter <int>            # Newton-Schulz iterations for SPECTRA (default 5)
```

### 5.3 μP implementation

μP (maximal update parametrization) has three coordinated components. Partial (items a + b) ships in Phase A; full (adds c) ships in Phase E.

#### (a) Init scaling rules (`halo_training/mup.py`)

For a model with hidden width `d` and `d_base` as the μP base width:
- Embedding layers: standard `N(0, 1)` init (no width scaling)
- Hidden weight matrices: `N(0, σ²/d_in)` with explicit fan-in scaling
- Readout (output projection / lm_head tied weight): `N(0, 1/d²)` scaling

Integrates with `OdinFlat._init_weights` via a flag check. The `d_base` defaults to 256 (matches our 30M probe's width).

```python
# halo_training/mup.py
def apply_mup_init(model, d_base=256):
    """Apply μP init scaling rules to an instantiated model."""
    for name, p in model.named_parameters():
        if p.dim() < 2:
            continue
        if "tok_embeddings" in name:
            continue  # embeddings stay at standard init
        d_in = p.shape[-1]
        # Fan-in scaling: std = 1/sqrt(d_in) is already PyTorch default;
        # μP adds a width-scale correction relative to d_base.
        scale_correction = (d_base / d_in) ** 0.5
        p.data.mul_(scale_correction)
```

Applied in `OdinFlat._init_weights` after Xavier+LN-scaling from Sprint 1:

```python
if self.use_mup:
    apply_mup_init(self, d_base=self.mup_base_width)
```

#### (b) Per-layer LR scaling (`halo_training/optimizer.py`)

Different parameter categories get different LR scaling under μP. The "base LR" here refers to the value passed as `--lr-2d` on the CLI (which is the peak hidden-layer LR in the Sprint 1 recipe; default 0.0235 per IMU-1). When `--mup` is active, the actual per-parameter LR is derived by μP's width-scaling rules rather than used verbatim.

| Category | LR multiplier (relative to `--lr-2d`) |
|----------|:-:|
| Embedding/input | × 1 (full LR) |
| Hidden 2D matrices | × (d_base / d_current) |
| Output (readout) | × (d_base / d_current)² |

For our OdinFlat at d=768 with d_base=256:
- d_ratio = d_current / d_base = 3
- Hidden LR: `--lr-2d / 3`
- Output LR: `--lr-2d / 9`

For the 30M probe at d=512 with d_base=256:
- d_ratio = 2
- Hidden LR: `--lr-2d / 2`
- Output LR: `--lr-2d / 4`

The 1D group (biases, LN gains, embeddings, lm_head tied weight) continues to use `--lr-1d` via AdamW exactly as in Sprint 1 — μP does not re-scale this group.

**Parameter group construction** (extends Sprint 1's splitter):

```python
def build_mup_param_groups(model, base_lr, d_base=256):
    """Build 3-way split for μP LR scaling."""
    d_current = model.d_model
    d_ratio = d_current / d_base
    
    emb_group, hidden_group, readout_group = [], [], []
    # ... classify each parameter into one of the three ...
    
    return [
        {"params": emb_group, "lr": base_lr},
        {"params": hidden_group, "lr": base_lr / d_ratio},
        {"params": readout_group, "lr": base_lr / (d_ratio ** 2)},
    ]
```

Used in `build_optimizer` when `--mup` flag is set. Replaces the Sprint 1 2-way split (2D vs 1D) with a 3-way split.

#### (c) Attention scale 1/d_head (Phase E, sequential)

Standard attention scores: `scores = Q K^T / √d_head`.
Full μP-Adam: `scores = Q K^T / d_head`.

Implemented in `NoPECodaAttention.forward`:

```python
def forward(self, x, ..., mup_attn_scale=False):
    # ... Q, K, V projection ...
    scale = 1.0 / self.head_dim if mup_attn_scale else 1.0 / math.sqrt(self.head_dim)
    scores = (q @ k.transpose(-2, -1)) * scale
    # ... rest of attention ...
```

`mup_attn_scale` set from `--mup-attn` CLI flag, defaulting to False. Phase E flips it to True for full-μP ablation.

## 6. Success criteria (Gate Sprint-1.5 → Sprint 3)

Sprint 1.5 succeeds if ALL of:

| Criterion | Target | Measured how |
|-----------|--------|--------------|
| **SPECTRA-post alone (C1) vs Sprint 1 baseline** | ≥ **1%** loss improvement | Factorial C1 vs S1-baseline, 2-epoch wikitext |
| **SPECTRA-pre marginal contribution (D1 vs C1)** | ≥ **0.3%** additional improvement | D1 vs C1 |
| **μP-partial alone (C2) vs Sprint 1 baseline** | ≥ **1%** loss improvement | C2 vs S1-baseline |
| **μP-full (E1) vs μP-partial (D2)** | ≥ 0.2% additional improvement (otherwise revert to partial-only) | E1 vs D2 |
| **SPECTRA + μP combined (D2 or E1)** | ≥ **1%** loss improvement (subadditivity tolerated) | D2 or E1 vs S1-baseline |
| **μP transfer claim** | 30M-derived `lr_2d` winner within ±10% of 122M optimum | E2, E3 sanity runs |
| **Throughput cost (combined recipe)** | ≤ **10%** additional tok/s cost on top of Sprint 1 | Steady-state measurement over 500+ steps |
| **Stability** | Zero NaN steps; max grad norm < 10 | StabilityGuard counters |

### Failure-response table

| Failure | Response |
|---------|----------|
| SPECTRA-post (C1) gain < 0.3% | Proceed to post+pre per plan; pre may carry most of the signal |
| SPECTRA post+pre (D1) < 0.5% total | Ship post-only (if C1 passed); drop pre; document null result |
| μP-partial (C2) regresses | Investigate init rules; retry with corrected LR; if persistent, fall back to IMU-1 LRs without μP init scaling |
| μP-full (E1) worse than μP-partial (D2) | Ship μP-partial (init + LR only); revert 1/d_head; document that kernel changes needed |
| μP transfer > ±10% off | μP transfer claim fails; ship μP parametrization anyway per brainstorm Q3.8; document the limitation |
| Combined gain (D2 or E1) < sum of individual parts | Subadditivity acceptable; still ship if combined > best individual |
| Throughput cost > 15% | Profile Newton-Schulz in SPECTRA; reduce iterations 5 → 3; if still bad, drop pre-clipping |
| LR probe all-diverge | Secondary probe at {0.010, 0.012, 0.015, 0.020}; if still divergent, μP incompatible at our arch, ship SPECTRA-only |
| NaN in any phase | Tripwire → pause, bisect, fix before continuing |

## 7. Execution phases

### Phase A — SPECTRA-post + μP-partial implementation (3 days dev)

**Implement:**
- `halo_training/spectra.py` post-clipping
- `halo_training/mup.py` partial μP (init + LR)
- `models/odin_flat_30m.py` proportional probe
- All CLI flags (both post and pre SPECTRA; both partial and full μP; default OFF)
- Unit tests (see §8)

**Exit criterion:** Unit tests pass; smoke test runs with all flags default-OFF (baseline parity); 1-step training with all-ON produces finite values.

### Phase B — Pre-ablation probes (~2 hours compute)

**B1: SPECTRA clip_norm sweep** — 3 runs, 200 steps each at 122M, `clip_norm ∈ {0.5, 1.0, 2.0}`. Select winner (lowest steady-state loss).

**B2: μP 30M LR probe** — 4 runs, 500 steps each at 30M, `lr_2d ∈ {0.015, 0.020, 0.0235, 0.030}`. Select winner (lowest steady-state loss).

**Exit criterion:** Both probes produce finite-loss winners. If winner is at a boundary, run a fallback extension probe.

### Phase C — Factorial with SPECTRA-post-only (~6 hours compute)

Three 2-epoch wikitext runs:

- **C1:** Sprint 1 + SPECTRA-post (clip_norm = B1 winner)
- **C2:** Sprint 1 + μP-partial (lr_2d = B2 winner scaled to 122M per μP)
- **C3:** Sprint 1 + SPECTRA-post + μP-partial

Each run: ~2 hours DDP on full wikitext 2 epochs. Total: ~6 hours sequential compute.

**Gate C → D:**
- C1 ≥ 1% improvement (SPECTRA-post alone)
- C2 ≥ 1% improvement (μP-partial alone)
- C3 ≥ 1% improvement (combined)

If any fails, apply failure-response table. Proceed to Phase D only if gate passes.

### Phase D — Add SPECTRA-pre (2 days dev + ~4 hours compute)

**Implement:** `halo_training/spectra.py` pre-clipping. Unit tests.

**Run:**
- **D1:** Sprint 1 + SPECTRA-post + SPECTRA-pre (no μP) — 2 epochs
- **D2:** Sprint 1 + SPECTRA-post + SPECTRA-pre + μP-partial — 2 epochs

**Gate D → E:**
- D1 ≥ 0.3% marginal over C1 (pre-clipping contributes)
- D2 ≥ 1.2% combined improvement over S1 baseline

If D1 fails the 0.3% marginal gate, drop pre-clipping from the shipped recipe (post-only survives).

### Phase E — Full μP + transfer sanity (2 days dev + ~4 hours compute)

**Implement:** 1/d_head attention scaling behind `--mup-attn` flag. Unit tests.

**Run:**
- **E1:** Sprint 1 + SPECTRA(post+pre) + μP-full (including 1/d_head attn) — 2 epochs
- **E2:** 122M with lr_2d = B2_winner × 0.8 (transfer sanity, half-epoch is enough, ~1 hour)
- **E3:** 122M with lr_2d = B2_winner × 1.2 (transfer sanity, ~1 hour)

**Gate E (Sprint 1.5 → Sprint 3 ready):**
- E1 ≥ D2 (full μP improves over partial)
  - If not: revert to μP-partial; do not ship 1/d_head
- E2, E3: 122M loss at ±20% LR should be worse than at B2_winner LR by ≥ 0.5% each, and the B2_winner LR's 122M loss should be within ±10% of the local minimum fit over the 3 points (E2, B2_winner, E3)
  - If transfer claim fails: ship μP parametrization without transfer-validation, document limitation

## Appendix A — Reference formulas

### SPECTRA post-clipping (composite Frank-Wolfe)

```
After optimizer produces ΔW:
  σ ← spectral_norm(ΔW)  via Newton-Schulz
  if σ > clip_norm:
    ΔW ← ΔW * (clip_norm / σ)
W ← W + ΔW  (apply clipped update)
```

### SPECTRA pre-clipping

```
After loss.backward() for each 2D parameter p with p.grad:
  σ_g ← spectral_norm(p.grad)  via Newton-Schulz
  if σ_g > clip_norm:
    p.grad.mul_(clip_norm / σ_g)
Then optimizer.step() proceeds as usual
```

### μP init scaling (Yang et al., applied via IMU-1 conventions)

```
For each 2D parameter W with shape (d_out, d_in):
  standard_init(W)                                 (Xavier or Kaiming)
  if not embedding:
    W ← W * sqrt(d_base / d_in)                    (μP width-scale correction)
```

### μP LR scaling

```
Base LR: lr_0  (from hyperparameter tuning or prior work)
d_ratio = d_current / d_base

For parameter groups:
  Embedding:   lr = lr_0
  Hidden 2D:   lr = lr_0 / d_ratio
  Readout:     lr = lr_0 / d_ratio²
```

### μP attention scaling (full μP, Phase E)

```
Standard:   scores = (Q K^T) / sqrt(d_head)
μP-Adam:    scores = (Q K^T) / d_head
```
